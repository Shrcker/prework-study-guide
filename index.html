HTML

  • The head element contains info about the page.
  • The body element represents the visible content of the page.
  • Sections are semantic groupings for thematic groups in the code.
  • Sections can have classes and id's assigned to them. Classes and id's can be defined in CSS later.
  • Classes and id's can also be manipulated by Javascript elsewhere!

CSS

  • CSS code is defined within curly brackets, { }, within a document.
  • CSS is good for organizing and creating the visual flairs of a page.
  • Classes and id's are usually defined by style sheets near the top of the document, within its own element.
  • CSS documents within projects define strict style sheets, allowing devs to make large aesthetic changes to a project while defining what components apply within HTML.
  • CSS code blocks begin with a selector followed by a declaration in curly brackets.
  • Declarations are sets of two key components: the CSS property we wish to apply to the selector, then its value separated by a semicolon.
  • Making a selector an asterisk makes the following delcarations apply to the whole HTML file.
  • CSS classes with a period proceeded by a selector (example: .card) which can then be applied to HTML sections by assigning them that class.

GIT

  • Git is a version control system used for organizing our projects non-linearly and collaborate with other developers.
  • Github is where we store our project repositories in a cloud format. It's essentially what hosts our Git projects.
  • With Git, we can easily track each version of a project and revert to specific states. It's useful for debugging and tracking our progress.
  • Anyone can collaborate with online Github projects.
  • We can also track known issues or bugs with project versions with Github.
  • One can remotely download and access files and repositories onto local work stations.
  • git status: checks what branch we're interacting with.
  • git checkout -b branch-name: creates a new branch and automatically switches to it.
  • git merge [branch]: merges the specified branch's history into the current one.
  • git log: shows all commits in the current branch's history.
  • git fetch [alias]: fetches down all the branches from that Git remote.
  • git push [alia] [branch]: transmits local branch commits to the remote repository branch.
  • git branch: lists your branches.a, * will appear next to the currently active branch.
  • git branch [branch-name]: creates a new branch at the current commit.
  • git rebase [branch]: apply any commits of current branch ahead of a specified one.
  • git reset --hard [commit]: clear staging area, rewrite working tree from specified commit.

Javascript

  • JavaScript allows us to add functionality for end user interaction with the webpage. Things like buttons, for instance.
  • There is a LOT to learn about JavaScript, so for more complete information, please read the official documentation as well.
  • To cover the Java used to present this site, let's go over Variables, Arrays, Iteration, and Functions.
  • Variables are pieces of data defined at the beginning of a JavaScript document. Mutable Variables are declared with either the var or let keyword, while immutable, or non-changing Variables are declared with const.
  • The difference in deciding to use let between var is scope. Variables declared with let are only available within the code block they're declared. Variables declared with var, however, are available within the Function they're defined.
  • Example: let myName = 5;
  • An important concept moving forward one should become familiar with are data types.
  • In very basic terms, JavaScript deals with four distinct data types: String, Numbers, Booleans, and Objects.
  • Think of these data types as distinct ways for the computer to understand our code, or instructions, to it. Knowing the distinction between data types helps define how to think in JavaScript and be able to more easily read it.
  • Arrays are ways to store data, however while Variables may only store one piece of data, an Array allows us to store data in a related list.
  • Example: let myCats = ['Ella', 'Misty', 'Pearl'];
  • Data can be retrieved from Arrays by calling for the data's position within the Array. Because we work with computers, the first item in an Array is item [0], with positions incrementing from the left thusly.
  • An example from the earler example Array would be that 'Ella' would be item [0]. You could use her in code elsewhere like: myCats[0];
  • Data in JavaScript can be expressed via labels, which enables flexibility within the language.
  • This is why we make Variables. With them, we can repeat complex or changing data throughout the code by using plain English, making the code easier to understand.
  • JavaScript is what we call "Object Oriented Programming," meaning that the code is divided and understood in discreet chunks of code and information that can speak to each other indirectly.
  • Classes are, usually, in-built (or baked) Objects. We can use them to call certain functionality into our code that's already been hard-coded into JavaScript. More experienced users may also create their own Classes.
  • Methods are usually children of Objects and can be called from them by separating the name of the Class and then the method with a period.
  • Example: console.log("Insert string here!");
  • Notice the String in parentheses. This is the argument of the Class's Method (the .log part of the example, basically it's the executable part of a Class) which lets us define values to apply to a Function, or the process of a Method.
  • Functions are kind of like mini-Classes, or are code blocks that can execute elsewhere in code.
  • Functions are created with the function keyword, followed by its name, then its argument which can be blank if you don't need to define specific value relationships. Then, between two curly brackets, we can write what code we want to execute when the function is called elsewhere.
  • Remember that Functions cannot be executed by themselves. Additionally, they become like Classes in that we can call them anywhere else now.
  • Example of calling a function: myFunc();
  • We can also loop code with for loops, which are, unsurprisingly, created with the for keyword. These use incremental math to loop a following line of code that may have its scope defined in its argument, having the loop stop once its scope is satisfied.
  • Example: for (var i = 0; i < topics.length; i++) { /*Insert Code Here.*/ }
